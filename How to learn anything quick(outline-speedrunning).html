Outline Speed-Running

How to learn something quickly and effectively

 

"Do 'outline speed running': Recursively outline an MVP, speedrun filling it in, and only then go back and perfect," user dnbt777 suggests on https://learnhowtolearn.org.

 

George Hotz, an American security hacker, entrepreneur, and software engineer, says everyone he's met who can program well learned it the same way: they had an idea and then built it.

They had something they wanted to do and then learned how to build it.

This is a fantastic mental model for learning anything quickly and in-depth.

 

Do you find yourself talented in design, music, or coding but unable to finish anything, stuck in perfectionism, and cursing yourself for not meeting your expectations?

 

This method is for you.

 

What is the outline speed run algorithm?

 

1. Create an Outline:

Draft an outline of the topic or project you want to learn or execute (the more application-oriented, the better).

2. Nested Outlines:

For each item in the outline, make a nested outline. Do this recursively until the items cannot get any smaller.

3. Start Small:

Begin working on the smallest units right away. Build momentum. Do not perfect as you go.

4. Fill in Larger Units:

Move on to the bigger units. Do this until all units are complete.

5. Refine the Final Product:

At this point, you'll have a rough final product. By now, you have built enough momentum to work further and get it finished. This will clear the clutter and leave you with enough mental bandwidth to think and design your project better. If you're learning a subject, this process helps you mentally organize the topics better, building strong, well-connected knowledge about the subject.

 

Summary:

 

>Build an Outline

> Speedrun: Fill in the items quickly. Do not stress about perfection.

>Do Not Perfect as You Go

> Refine: Go back to the items and improve them until you are satisfied with the finished product.

 

Application of this methodology

My Usual Way of Making Music:

 

a. Initiate when a random inspiration strikes.

b. Start making the song from left to right (time progression) while deciding the composition, sounds, and mixing all at once.

 

This is why I can't finish my work.

 

Using outline speedrunning:

 

Step 1>Outlining the sketch

 

I’ll sit and think about what I want the track to sound like.

While thinking about that, I’ll decide what carries the main characteristic of the track.

For example:

 

I'm making a hip-hop track, and I want the drum pattern to be the center of the entire track (for the maximum number of bars at least).

 

{

    Outline for the drums and arrangement:

    {

        Kick(s): I'll collect the sounds I intend to use for kicks.

        {

            I'll either use pre-made one-shot samples or sound design my own.

        }

        Snares:

        {

            I'll collect all sounds that might work as a snare.

        }

        Other percussion:

        {

            Foleys, recordings, tabla, sound design one-shots, etc.

        }

    }

   

    Making a drum pattern:

    {

        Design a pattern. 

        You are not supposed to use all sounds at once.

        Pick a specific few sounds and make a pattern.

        Place it in the timeline.

        Make a drum-only arrangement.

    }

}

 

Step 2> Going back recursively and making changes

By now, I am left with a super simple arrangement. Then, I'll keep going back recursively, changing the sounds, making changes in the arrangement, etc.

 

a. Change the sound of the snares, kicks, or use all of them separately in different places.

b. Time-stretch a few sounds to tweak them.

c. Add a lot of mixing fx VSTs.

d. Use granular synthesis to make more layers.

e. Add more layers.

f. Change the composition while adding more sounds and instruments.

g. Change the tempo.

h. Change the drum pattern at specific places.

 

The list is infinite and endless.

 

Conclusion

 

This same methodology applies everywhere. Go on with your life using it right away.
